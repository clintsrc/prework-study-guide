HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Best practice for using header elements is to begin with h1 and add them sequentially h2, h3, etc. for accessibility
  • Items (metadata) in the head section indicate the page's language and character set of the page, http headers, default view and the page's title caption
  • Within the page's body, common elements are: header, main and footer
  • The header element is useful for navigation menus and introductory content
  • The main element contains the (javascript) code
  • The footer element is used to show author, copyright, contact, sitemap, and navigation
  • Other tags for the body include things such as img (images), a (link anchors), ol/ul (ordered or unordered lists) and li (list items)
  • Each element can have attributes to provide even more specific characteristics like a unique id or a modified style (class)

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS: apply a style attribute to an individual HTML tag (the least flexible)
  • Internal style sheet: an embedded style sheet defined with style elements in the head section. These defined rules only apply to elements on that page
  • External CSS style sheet: CSS rules are defined in a separate file. A change to that file applies for all pages that use it (the most flexible and most common). A link element located in the head section points to the css file
  • CSS box model: Every HTML element is essentially a box. Each box consists of Margins (space outside the element), the element's Borders, Padding (space around the content inside an element), and the Content itself
  • display: block;
    the element starts on a new line
  • For the example: 
    div {color: blue; text-align: justify;}
    div is the selector: the selector is one or more elements (comma separated) to which the rule applies
    Declaration block: delimited by braces, it contains one or more declarations
    Declaration: defines a 'property: value' set of characteristics to apply to the selector, each declaration is terminated by a semicolon
  • class selector: lets you define a rule that you can apply as a class attribute to specific elements
    .classname means apply the style to any element that has the classname attribute

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone git@github.com:org/repo.git: pulls down a repository from githhub to your local system via ssh
  • git add -A: stage all changed files locally to be checked in
  • Egit commit -m "description": commit staged changes with a comment
  • git pull origin main: update local files to with any recent changes in the remote repository
  • git push origin branch-name: push your locally committed changes to the remote repository
  • eval "$(ssh-agent -s)": when using ssh access, start the ssh agent
  • ssh-add ~/.ssh/privatekey.pem: when using ssh access,add your private key to the agent
  • ssh-add -l: when using ssh access, list the keys registered to the agent for github access
